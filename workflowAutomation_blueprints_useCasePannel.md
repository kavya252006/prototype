# workflowAutomation_blueprints_useCasePannel.html

This file is the filter sidebar from the Blueprints catalog page — not individual use cases, but the **category labels** used to filter all 174 blueprints. The number after each one is how many blueprints fall into it (a blueprint can belong to more than one category, so the numbers add up to more than 174).

## Use Case Categories

- **Advanced Reporting** (4) — Blueprints that build reports or dashboards, like sending a service scorecard to its owner.
- **Alert Triage & Routing** (4) — Blueprints that sort incoming alerts and send them to the right person or team.
- **Automate Manual Tasks** (8) — Blueprints that take a repetitive manual job (like restarting a service) and make it a one-click or automatic action.
- **Developer Experience** (18) — Blueprints that make life easier for developers, like scaffolding a new service or nudging someone about a stale merge request.
- **Governance** (7) — Blueprints that enforce rules, like requiring every service to have an owner tag or an assigned tier.
- **How To** (4) — Short guides teaching you how to use workflow features like variables, loops, and datastores.
- **Incident Automation** (7) — Blueprints that automatically do things when an incident happens, like creating a Jira ticket or a DORA failure record.
- **Incident Response & On-Call** (21) — Blueprints that help teams respond faster to incidents: paging the right people, posting updates, and following up afterward.
- **Onboarding** (2) — Blueprints that help set up new users, teams, or services.
- **Remediation** (19) — Blueprints that automatically fix a problem once it's detected, like restarting a broken service or turning off a bad feature flag.
- **Security Identity Protection** (37) — Blueprints that manage user accounts and access: suspending, locking, or cleaning up accounts in tools like Okta, Auth0, and Google Workspace.
- **Security Threat Containment** (12) — Blueprints that isolate or block a threat once it's found, like quarantining an infected device.
- **Security Threat Intelligence** (19) — Blueprints that check whether something (an IP, URL, file, etc.) is known to be dangerous, using outside threat databases.
- **Security Threat Investigation** (6) — Blueprints that gather more information about a potential threat to help an analyst dig deeper.

## Datadog Products Filter

This second filter group lets you narrow blueprints down by which Datadog product they connect to:

- **Case Management** (1) — Works with Datadog's case tracking feature.
- **Internal Developer Portal** (19) — Works with Datadog's service catalog and developer portal.
- **Monitors** (20) — Works with Datadog alerting/monitors.
- **Scorecards** (6) — Works with Datadog's service scorecards (best-practice grading).
- **Security** (101) — Works with Datadog's security products; this matches the count in `soarWorkflows.html`.
- **Software Catalog** (9) — Works with Datadog's software catalog of services.

Note: the sidebar also shows an "All Blueprints" total of 174 and a "New" filter with 1 result, but those aren't use-case categories — they're just navigation shortcuts.
